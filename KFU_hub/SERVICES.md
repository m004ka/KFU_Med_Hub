# Микросервисы KFU Hub Med

Интеграционная платформа медицинской санчасти КФУ состоит из 8 микросервисов.

---

## kfu-hub-gateway — API Gateway
**Порт:** `8080`

Единая точка входа для всех клиентов платформы.

- Маршрутизация запросов к нужному микросервису
- Валидация JWT-токенов перед проксированием
- Rate limiting через Redis (защита от перегрузки)
- Circuit breaker — fallback-ответы при недоступности сервисов
- CORS-политики для фронтенда

---

## kfu-hub-auth — Auth Service
**Порт:** `8081`

Управление пользователями, ролями и аутентификацией.

- Регистрация и авторизация пользователей (врач, студент, преподаватель, исследователь, администратор)
- Выдача и обновление JWT access/refresh токенов
- SSO-интеграция с корпоративной системой КФУ через Keycloak (OIDC/SAML)
- Двухфакторная аутентификация (2FA) для ролей врача и администратора
- Делегирование прав и временное расширение доступа с аудит-логированием

---

## kfu-hub-dataset — Dataset Service
**Порт:** `8082`

Центральный каталог и хранилище медицинских датасетов.

- Загрузка датасетов (CSV, JSON, XLSX, DICOM, NIFTI, FHIR Bundle, Parquet) до 5 ГБ в MinIO
- Управление метаданными: название, теги, источник, лицензия, схема данных
- Полнотекстовый поиск и фасетная фильтрация через Elasticsearch
- Превью первых 50 строк и статистические сводки при просмотре
- Версионирование датасетов с историей изменений
- Автоматическая проверка качества: дубли, пустые значения, выбросы
- Гранулярный доступ: публичный / групповой / приватный

---

## kfu-hub-ai-gateway — AI Gateway Service
**Порт:** `8083`

Управление очередью задач ИИ-анализа и интеграция с ML-моделями.

- Приём и постановка задач ИИ-анализа в очередь Kafka
- Real-time статус выполнения задачи через SSE (Server-Sent Events)
- HTTP-интеграция с NVIDIA Triton Inference Server (ONNX, GPU)
- HTTP-интеграция с Hugging Face API (NLP: ruBERT, MedBERT-ru)
- Хранение истории запусков и результатов анализа
- Экспорт результатов в PDF, CSV, JSON
- Поддерживаемые сценарии: классификация МКБ-11, анализ рентгена, NLP историй болезней, лабораторные данные, предиктивная аналитика рисков, сегментация МРТ/КТ

---

## kfu-hub-integration — Integration Service
**Порт:** `8084`

Адаптеры для интеграции с внешними медицинскими системами.

- HL7 FHIR R4 сервер и клиент (HAPI FHIR) — обмен историями болезней, назначениями, результатами анализов
- Валидация FHIR-ресурсов по профилям Минздрава РФ
- DICOM WADO/STOW/QIDO адаптер (dcm4che) — приём и передача рентген-снимков, МРТ, КТ
- REST-мост для интеграции с республиканскими МИС
- SOAP-адаптер для подключения к унаследованным государственным ИС

---

## kfu-hub-projects — Projects Service
**Порт:** `8085`

Управление исследовательскими проектами и задачами.

- Создание исследовательских проектов с участниками разных ролей (врач + преподаватель + студент)
- Публикация задач от медицинского персонала (описание, компетенции, дедлайн)
- Kanban-доска для управления статусами задач внутри проекта
- Файловое хранилище проекта в MinIO с историей версий
- Привязка датасетов и результатов ИИ-анализа к проектам
- Публичный каталог открытых задач для всех пользователей платформы

---

## kfu-hub-notifications — Notifications Service
**Порт:** `8086`

Доставка уведомлений пользователям платформы.

- Kafka consumer — подписка на события от всех сервисов (новые задачи, смена статуса, комментарии, завершение ИИ-анализа)
- Email-уведомления через SMTP (шаблоны на русском языке)
- In-app уведомления в реальном времени через Redis pub/sub
- Настройки предпочтений уведомлений для каждого пользователя

---

## kfu-hub-audit — Audit Service
**Порт:** `8087`

Журнал действий пользователей для соответствия требованиям 152-ФЗ.

- Kafka consumer — приём аудит-событий от всех сервисов платформы
- Хранение полного журнала: кто, что и когда сделал
- Партиционирование таблицы по месяцам для производительности
- Автоматическое создание новых партиций по расписанию
- Ретенция данных — 3 года (требование 152-ФЗ о персональных данных)
- REST API для поиска, фильтрации и экспорта журнала (доступно администратору)
